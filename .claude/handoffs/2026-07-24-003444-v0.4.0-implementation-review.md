### 1.2 Five Blind Spots (from previous handoff)

| 盲区 | 严重度 | 说明 |
|:---|:---:|:---|
| 相对路径引用 | 🔴 | `Path("app_control.py")` — CI 炸的根源 |
| 测试→源文件一致性 | 🟡 | 测试中硬编码路径在重构后断裂 |
| 模块级 import 断裂 | 🟡 | 项目内模块被移动后 import 静默失效 |
| env 样式不一致 | 🔵 | `os.environ["KEY"]` 等变体漏扫 |

## 三、关键实现细节与修正

### 3.1 T1: `file_refs.py` — 相对路径引用扫描

**依据**：handoff 盲区 1 的正则草稿 + 高级模型的设计决策。

**实现**：
- `QUOTED_REF_PATTERN`: 匹配 `Path("xxx.py")` / `Path('xxx.json')` / `open("xxx.csv")` / `open('xxx.txt')`
- `SCRIPT_REF_PATTERN`: 匹配 `subprocess.Popen(["python", "script.py"])` 等脚本引用
- `ALLOWED_EXTS`: 白名单过滤（`.py/.json/.yaml/.yml/.csv/.txt/.md/.toml/.cfg/.ini/.conf/.xml`）
- 存在性校验：`(root / ref_path).exists()` — 天然防误报
- 行号提取：`content[:m.start()].count('\n') + 1`
- 去重：`(line_num, ref_path)` tuple 上的 `seen_keys`

**与草稿的偏差**：
- 原草稿的 `REL_PATH_PATTERN` 用 `(?:Path|open)` 分组统一处理，实现采用了具名分组 `(?P<type>Path|open)(?P<path>...)`，方便后续区分引用类型
- 原草稿只提了 `Path()` 和 `open()`，子 agent 额外实现了 `SCRIPT_REF_PATTERN` 覆盖 subprocess 模式（这是合理的增量）
- 去重逻辑原草稿未提及，子 agent 自行补充

**需要审查的点**：
- 正则是否过于宽松（是否匹配了非文件路径的 `Path("model_name")`？答：白名单过滤了无扩展名的裸字符串）
- `SCRIPT_REF_PATTERN` 使用 `re.VERBOSE` 模式，注意空格不能出现在正则体中
- 去重策略是否足够（tuple key 包含行号 + 引用路径）

### 3.2 T2: `local_graph.py` — import 目标存在性回查

**依据**：handoff 盲区 3 + 高级模型"独立但互补"裁决。

**实现**：
- 在 `run()` 函数的 import 匹配循环中，对每个选中（`chosen`）的本地模块引用做 `(root / local_modules[chosen]).exists()` 回查
- 文件存在 → 加入 `local_dep_graph`（原行为）
- 文件不存在 → 加入 `broken_imports`（新行为）
- 输出扩展：`{"local_dep_graph": {...}, "broken_imports": {"file.py": ["module_x"]}}`

**需要注意的设计决策**：
- 只回查被匹配为本地 IMport 的目标，不额外扫描全量 import
- `broken_imports` 是新的顶级 key，与 `local_dep_graph` 同级（不嵌套），以便 CLI 的 `_check_broken_refs()` 统一扫描
- 父包无 `__init__.py` 的情况也被正确识别为断裂（`_collect_local_modules` 中注册了父包路径）

### 3.3 T3: `env_vars.py` — 环境变量变体

**依据**：handoff 盲区 4。

**实现**：
- `PYTHON_ENV_PATTERNS` 新增 3 个正则：
  - `environ.get("KEY")` — 不带 `os.` 前缀
  - `environ["KEY"]` — 不带 `os.` 前缀
  - `(?:os\.)?environ.setdefault("KEY", "val")` — setdefault 变体
- `required`/`default` 判断条件增加 `'setdefault' in line`，确保 setdefault 的默认值被正确提取

**需要注意的设计决策**：
- 当前 `PYTHON_ENV_PATTERNS` 已经包含 `os.environ["KEY"]`（第 18 行），所以此变体无需重复添加
- `environ["KEY"]` 没有默认值，`required: true`
- `environ.setdefault("KEY", "val")` 能正确提取默认值

### 3.4 T4: `--strict` CLI 标志

**依据**：高级模型的"默认 warn，`--strict` 时 exit 1"裁决。

**实现**：
- 新增 `_check_broken_refs(result, strict)` 函数，扫描两个来源：
  1. `file_refs` 模块结果中 `exists: false` 的条目
  2. `local_graph` 模块结果中 `broken_imports` 条目
- 默认模式（无 `--strict`）：stderr 输出 `[WARN]`，exit 0
- `--strict` 模式：stderr 输出详情，`sys.exit(1)`
- 不改变 `--format json` 和 `--modules` 的现有行为

**需要审查的点**：
- `_check_broken_refs` 在 `main()` 最后调用，是否覆盖了所有退出路径（json/plain 两种格式都走）
- 断裂引用汇总的输出格式是否清晰

### 3.5 已知需告警的问题

**pre-existing bug**：`model_refs.py` ��� `format_plain()` 中 `f['count']` 对字符串类型报错（`test_print_plain` 失败）。根源是 `format_plain` 在遍历 `model_refs` 时假设 `model_info` 是 dict，但实际某些情况下 `model_info` 可能是字符串。此问题在 v0.3.1 中已存在，**本次未触及**。
